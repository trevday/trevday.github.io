Flow Field Pathfinding (2018)

GitHub

This is a project I wrote for a class in Game AI. The idea with this project was to implement a version of flow field pathfinding as described in this paper. Java with Processing was possibly not the best choice for performance, but it offers a simple library for visual output and I wanted to prototype quickly without worrying about those details. I believe this flow field implementation is pretty successful, and the "extra features" such as formations work well, but individual unit steering does leave something to be desired. That, however, was not the focus of the project, and is really here to aid in providing a visual demonstration of the algorithm.

Some Quick Features:
  • Flow field pathfinding which subdivides the map into "sectors" of a configurable size
    • A per sector flow field cache that is shared amongst all units, which means units navigating in a similar manner through a given sector can use the same calculated flow field
    • A* search to chain sectors for unit navigation
  • Navigation optimizes for differences in terrain "height", or difficulty of moving through that terrain
    • Also supports navigating around impassable obstacles
  • A formation system that allows grouping arbitrary numbers of units together with a specified separation pattern
    • The formation system is very modular, and adding new formations requires a single new function
    • Formations are dynamic and will reform correctly if units are added or removed
  • Units have a very simple collision avoidance behavior that sends out a single raycast and applies an avoidance force if the unit will collide with something
